Key Algorithm

Shared Backbone · 1회 Encode

추론 시 backbone을 한 번만 실행하고, 동일 feature map을 foot / wound / dfu head가 공유합니다.

왜 중요한가

ViT backbone이 연산의 대부분을 차지하므로, multi-task에서도 latency를 최소화하면서 head를 독립적으로 교체·재학습할 수 있습니다.

backbone 1× head 독립 checkpoint task별 재학습 
# models/pipeline_model.py
class DFUPipelineModel(nn.Module):
    def encode(self, x):
        return self.backbone(x)  # 1회

    def predict_foot_logits(self, features, ...):
        return self.foot_head(features)

    def predict_wound_logits(self, features, ...):
        return self.wound_head(features)

    def predict_dfu_logits(self, features):
        return self.dfu_head(features)

Key Algorithm

Gate Logic + Foot Feature Crop

Inference Gate

Foot: area ratio ∈ [0.08, 0.5]
Wound: foot 탐지 + 중앙 ±0.25
DFU: foot 탐지 시 실행

Wound Crop

foot mask bbox를 margin(0.15)만큼 확장한 feature crop만 wound head에 전달 → 배경 노이즈 감소, 국소 정밀도 향상

# inference/pipeline.py (요약)
def predict_wound_logits_for_foot_crop(...):
    bbox = bbox_from_mask(foot_mask, margin)
    crop = features[..., y0:y1, x0:x1]
    logits = model.predict_wound_logits(crop, size)
    full = fill_neg_inf(model_size)
    full[..., y0:y1, x0:x1] = logits
    return full

# DFU head: spatial mean pooling
pooled = features.mean(dim=(2, 3))
return mlp(pooled)  # dfu / other
